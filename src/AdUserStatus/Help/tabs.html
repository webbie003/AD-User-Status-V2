Understanding Tabs & Results

Understanding User Result Tabs

When you process an input file, the application categorizes each user record into one of four tabs. These tabs help you quickly understand account status and where further action may be required.

Enabled
Users found in Active Directory and currently active. These accounts can sign in and access services as normal.
Disabled
Users found in Active Directory but currently disabled (e.g., departures, suspended access, inactive system accounts).
Not Found
The email or username could not be matched to an account in Active Directory. This may indicate a typo or that the account has been deleted.
External
Addresses outside the organization’s internal domains—typically external contacts, contractors, or partners not managed by your domain.

Information: Tab headers will update automatically as the Check User process runs which will display a realtime count (e.g., Enabled (24)) for each category.

Tips: Right-click a row to Copy Email or Copy Row if you only need details for a single user, otherwise select Export Results once the Check User process completes to consolidate all tabs into a single Excel file for recordkeeping.

How Internal Domains Are Identified

During the initial LDAP or LDAPS connection, the application queries Active Directory for domain information to determine which email addresses belong to the organization. This discovery ensures that user accounts are correctly classified as Internal (within trusted domains) or External (outside the organization).

The internal domain suffixes are retrieved directly from the connected directory’s configurationNamingContext by enumerating crossRef objects in the CN=Partitions,CN=Configuration,DC=... container. Each nCName value represents a recognized internal DNS namespace.

  • Each discovered suffix (e.g. example.com, example.local) is stored temporarily in memory.
  • Email domains from the input file are compared against this list.
  • If a match is found, the user is considered Internal.
  • If no match is found, the user is marked as External.

Tip: This method allows the app to adapt automatically to multi-domain or hybrid environments without needing manual configuration of domain names.

Information: For troubleshooting, you can manually verify the same information in Active Directory using:
ldp.exe → Connect to a DC → Browse → Configuration → Partitions → crossRef objects.
