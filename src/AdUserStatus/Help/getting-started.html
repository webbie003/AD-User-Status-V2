ADUserStatus

Getting Started

What You Need

  • Access to your organisation’s Active Directory (Windows integrated authentication).
  • Network reachability to a domain controller (LDAPS 636 preferred, LDAP 389 fallback).
  • An input file that includes a header row with the following fields:
    • EmailAddress (required)
    • SamAccountName (optional)
    • DisplayName (optional)

    Information: The file may contain additional columns — only the EmailAddress field (or one of its recognised aliases) is required for the Check User process to run successfully.

Input File Example

You can include other information in your spreadsheet if needed. The application will ignore unrelated columns during processing and focus on matching users by EmailAddress or SamAccountName.

Information: When checking users, the application first searches Active Directory using the mail attribute. If no match is found, it then checks the userPrincipalName (UPN) and any aliases listed in proxyAddresses. This ensures alternate or secondary email addresses are recognized automatically.

Basic workflow

  1. Open the app and wait for the connection status to show LDAPS (or LDAP if LDAPS isn’t available).
  2. Click Browse (or drag & drop) your file. The app will show a “Ready” callout.
  3. Click Check User to process. Tabs update automatically as the Check User process runs.
  4. Review results in the four tabs (Enabled, Disabled, Not Found, External).
  5. Click Export Results to save an .xlsx of all tabs.

Tip: Right-click a row to Copy Email or Copy Row to the clipboard.

Quick Troubleshooting

  • No results? Confirm the input spreadsheet contains a header of EmailAddress or Email (case-insensitive).
  • Connection shows only LDAP (insecure)? Check LDAPS (636) reachability and DC certificate.
  • External vs Not Found? External means the domain isn’t one of your internal domains.
