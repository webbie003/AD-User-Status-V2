Security & Data Handling

Authentication & Transport

  • Authentication: Windows Integrated (Negotiate/Kerberos/NTLM) via the OS context; credentials are not stored by the app.
  • Transport: The app prefers LDAPS (636) with TLS. If unavailable, it falls back to LDAP (389).

Data handled

  • Inputs: email addresses and optional account identifiers from your file.
  • Queries: directory lookups for existence and enabled/disabled state.
  • Outputs: an .xlsx export of the results (Enabled/Disabled/Not Found/External).

Warning: No passwords or secrets are collected or used by this application, avoid including any unnecessary PII in your input files.

Recommended controls

  • Enforce LDAPS; disable simple binds on LDAP where possible.
  • Enable LDAP channel binding and LDAP signing on domain controllers.
  • Use least-privilege directory permissions for the user context running the app.
  • Validate DC certificates (trusted CA, correct EKU, not expired).
  • Secure exports: store the generated .xlsx on trusted locations; apply access controls.

Logging & privacy

  • Operational messages are displayed in-app; errors are shown to the user.
  • Avoid placing exported files in public/shared folders unless required.
